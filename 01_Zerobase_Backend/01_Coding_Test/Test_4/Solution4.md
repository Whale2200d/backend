# Test 4

## 문제 설명

지우는 운영체제 수업에 푹 빠져, 자기만의 운영체제를 제작하기로 마음먹었다.
지우는 현재 CPU의 작업 스케쥴러를 구현하고 있다.

지우가 구현하고자 하는 CPU의 작업 스케줄러는 아래와 같이 설계되어 있다.

- CPU가 처리할 작업이 없으면 IDLE 상태를 유지한다.
- CPU가 처리할 작업이 있는 경우, 가장 처리 시간이 짧은 작업을 선택하여 처리한다.
- CPU는 하나의 작업이 끝날 때까지 멈추지 않고 처리한다.
- CPU가 하나의 작업을 모두 마친 후에 다른 작업을 시작할 수 있다.

처리할 작업은 아래 입력으로 주어진다.

- `start` : 각 처리할 작업이 CPU에 주어진 시각이 담긴 배열
- `time` : 각 작업을 처리하는 데에 걸리는 시간이 담긴 배열

이 때, CPU가 처리하는 작업의 순서를 배열 형식으로 출력하라.
(단, 처리 시간이 동일한 작업이 동시에 처리 가능한 경우, 더 낮은 인덱스를 가진 작업을 선택한다.)

## 입력 설명

- `0 < start.length = time.length <= 1000`
- `0 <= start[i] <= 100000`
- `1 <= time[i] <= 1000`

## 출력 설명

- CPU가 처리하는 작업의 인덱스 순서를 담은 배열(0-based 인덱스)

## 입출력 예시

- 입력
  - `start = {0, 2, 3, 5, 6}`
  - `time = {2, 4, 2, 1, 3}`
- 출력
  - `{0, 1, 3, 2, 4}`
